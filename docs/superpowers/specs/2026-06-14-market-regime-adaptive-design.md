# 市场自适应参数系统 v6 — 设计文档

> 日期: 2026-06-14
> 状态: ✅ 已实现
> 关联: [[simpsons-paradox]]

## 问题

A股连板回调策略在不同市场环境下表现差异巨大。当前 v5 使用统一参数，在 2023.01-2024.06 震荡下行期（Period A）夏普 -2.08、胜率 42%，严重亏损。能否通过市场状态自适应切换参数来改善？

## 方法

### 1. 三时段独立深度优化

对三个市场阶段分别跑增强版三阶段网格搜索（~7.4万组合 Stage 1 → 8万组合 Stage 2 → 5万组合 Stage 3）：

| Period | 时间 | 市场特征 |
|--------|------|----------|
| A | 2023.01 - 2024.06 | 震荡下行（熊市） |
| B | 2024.07 - 2025.06 | 924行情+牛市大涨 |
| C | 2025.07 - 2026.04 | 震荡回调 |

### 2. 3×3 交叉验证矩阵

用每个时段的最优参数，在另外两个时段的事件上回测，形成 3×3 矩阵。

### 3. 市场状态检测

基于三大指数（上证/深证/创业板）5日趋势，分5档情绪：
- 1档 冰点期（< -2%）→ bear
- 2档 低迷期（-2% ~ -0.5%）→ bear
- 3档 启动期（-0.5% ~ +1%）→ 非熊市
- 4档 发酵期（+1% ~ +3%）→ 非熊市
- 5档 高潮期（> +3%）→ 非熊市

## 结果

### 各时段最优参数

| 参数 | A (熊市) | B (牛市) | C (震荡) | 当前统一 |
|------|:------:|:------:|:------:|:------:|
| pullback_ratio_min | **0.06** | 0.15 | 0.01 | 0.12 |
| pullback_ratio_max | **0.11** | 0.22 | 0.32 | 0.40 |
| volume_shrink_ratio | **0.41** | 0.40 | 0.84 | 0.67 |
| hold_days | **7** | 10 | 11 | 10 |
| take_profit | 0.057 | 0.06 | 0.052 | 0.051 |
| stop_loss | -0.103 | -0.12 | -0.144 | -0.112 |
| min_consecutive_limit_up | 2 | 2 | 3 | 3 |

### 3×3 交叉验证矩阵

| Params↓ Test→ | Period A (熊) | Period B (牛) | Period C (震荡) |
|---------------|:------------:|:------------:|:---------------:|
| **A参数 (熊)** | ✅ 夏普 **7.1** WR 51% | 夏普 8.0 WR 60% | 夏普 5.1 WR 49% |
| **B参数 (牛)** | ❌ 夏普 **-3.7** WR 20% | ✅ 夏普 12.1 WR 79% | ✅ 夏普 **14.6** WR 46% |
| **C参数 (震荡)** | ❌ 夏普 **-1.1** WR 51% | 夏普 5.1 WR 74% | ✅ 夏普 8.0 WR 66% |

### 关键发现

1. **熊市是根本不同的问题**：统一参数（B/C）在 Period A 的夏普分别为 -3.7 和 -1.1，而熊市专属参数（A）为 +7.1。差距不是"调参"级别，而是"完全不同的策略逻辑"。

2. **A参数的核心差异**：回调上限从 40% 砍到 11%。熊市中，股票从涨停高点回调超过 11% 基本就是在继续崩，不是反弹。配合极度缩量（41%）和缩短持有（7天），实现了"浅回调+快进快出"的熊市打法。

3. **牛市/震荡可以共用一套参数**：B参数在 C 时段甚至比 C 自身参数更好（夏普 14.6 vs 8.0）。这意味着只需要两套参数：熊市 vs 非熊市。

4. **自适应 vs 统一的量化对比**：
   - 统一参数（B）跨三期均夏普：7.7
   - 自适应参数跨三期均夏普：**11.3**（提升 +3.6）

### 集成点

- **`选股new_v5.py`**: `screen_today()` 默认 `mode='auto'`，自动检测→切换
- **`auto_daily.py`**: 每日定时扫描加入 bear 模式 + 市场状态记录

### CLI 用法

```bash
python 选股new_v5.py --today          # 自动检测市场，选择最优模式
python 选股new_v5.py --today auto     # 同上
python 选股new_v5.py --today bear     # 强制熊市模式
python 选股new_v5.py --today strict   # 强制严格模式
python 选股new_v5.py --today loose    # 强制宽松模式
```

## 使用建议

| 市场状态 | 检测条件 | 自动模式 | 信号特征 |
|----------|----------|----------|----------|
| 🐻 熊市 | 5日均趋势 < -0.5% | bear | 信号少（~3/月），质量高（50%胜率），浅回调 |
| 🐂 牛市 | 5日均趋势 > 1% | loose | 信号多（~10/月），高胜率（75%+） |
| 📊 震荡 | -0.5% ~ 1% | strict | 信号适中，高质量（70%胜率） |

## 局限性与后续

1. **Period B 参数仅经过 Stage 1**：因牛市事件量巨大（38,112个），完整三阶段优化耗时过长。建议在后台补跑 Stage 2+3。
2. **情绪档位是简化模型**：仅基于5日趋势，未纳入成交量、波动率等维度。
3. **未做 walk-forward**：交叉验证使用的是事件级评估（`evaluate_params_on_events`），非真实逐日前向回测。建议在积累更多数据后做 walk-forward 验证。
4. **冰点期（1档）**：avg_trend < -2% 时理论上应完全空仓，当前仍会产生少量 bear 模式信号。可考虑增加"完全空仓"选项。
